How to clone & push to and from the test repository

When you create a repository on GitHub, it exists as a remote repository. You can clone your repository to create a local copy on your computer and sync between the two locations. If it belongs to anyonelse you can use the fork option links above.

HOW TO CLONE FROM THE TEST REPOSITORY

To clone a repository use following command:"git clone https://github.com/renes/fh-test-repo"

Validation: Go to the local repository- project folder use following command: "Git status"

HOW TO PUSH THE TEST REPOSITORY

To clone a repository use following command: "git push"

Validation: After you’ve pushed your commits, visit your repository on GitHub and notice that your changes are reflected there, and also that you have access to the full commit history for your repository!

How to start a docker container

If the container already exist. You can start your docker container with the following command "docker start [OPTIONS] CONTAINER [CONTAINER...]"

Forexample: "$ docker start my_container"

if you want to start a container from an image used the following command: docker run [OPTIONS] IMAGE [COMMAND] [ARG...]

Forexample $ docker run --name test -it debian

Validation: "Sudo docker ps" or "sudo docker container ls"

How to expose a docker port

If you want to expose a container, use the command: " docker run -i --expose=network_name container_name bash"

Forexample "$ docker run -i --expose=80 b5593e60c33b bash" or you can use the command with multiple -p option for port mapping as like: "docker run -d -p 80:80 -p 5802:5802 .....""

Validation: "Sudo docker ps" or "sudo docker container ls"

How to stop a docker container

You can stop your docker container with the following command " docker container stop [OPTIONS] CONTAINER [CONTAINER...] "

Forexample: "$ docker container stop my_container"

Validation: "Sudo docker ps" or "sudo docker container ls"
